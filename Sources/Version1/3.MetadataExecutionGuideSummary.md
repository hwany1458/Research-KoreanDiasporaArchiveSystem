## 📚 3단계 메타데이터 통합 계층 실행 가이드 요약

### 1. schema.py (메타데이터 스키마)

|클래스 | 기능 |
|---------|----------|
|`DublinCoreMetadata` | Dublin Core 15개 핵심 요소 |
|`DiasporaExtension` | 디아스포라 확장 (이주, 인물, 장소, 행사) |
|`PersonInfo` | 인물 정보 |
|`LocationInfo` | 장소 정보 |
|`EventInfo` | 행사 정보 |
|`UnifiedMetadata` | 통합 메타데이터 |
|`MetadataFactory` | 메타데이터 생성 팩토리 |

-----

사용 예시:
```
from src.core.metadata.schema import MetadataFactory, MaterialType

# 이미지 메타데이터 생성
metadata = MetadataFactory.create_for_image(
    title="1975년 가족 사진",
    file_path="photo.jpg",
    width=1200, height=800
)

# JSON 변환
json_str = metadata.to_json()
```

### 2. relation_engine.py (관계 추론 엔진)

|클래스 | 기능 | 기술 |
|---------|----------|----------|
|`FaceRelationEngine` | 얼굴 기반 인물 연결 | face_recognition |
|`TemporalRelationEngine` | 시간 기반 관계 | 정규식 |
|`SpatialRelationEngine` | 장소 기반 관계 | 지명 정규화 |
|`TopicRelationEngine` | 주제 기반 관계| sentence-transformers |
|`RelationInferenceEngine` | 통합 엔진 | 전체 통합 |

-----

사용 예시:
```
from src.core.metadata.relation_engine import RelationInferenceEngine

engine = RelationInferenceEngine({"device": "cpu"})
engine.load_models()

# 관계 추론
relations = engine.infer_relations(items)

# 관련 자료 조회
related = engine.get_related_items(item_id)
```

### 3. knowledge_graph.py (지식그래프)

|클래스 | 기능 |
|---------|----------|
|`Node` | 그래프 노드 (Item, Person, Location, Event, Topic) |
|`Edge` | 그래프 엣지 (관계) |
|`KnowledgeGraph` | Neo4j 연동 + 인메모리 폴백 |

-----

사용 예시:
```
from src.core.metadata.knowledge_graph import KnowledgeGraph

kg = KnowledgeGraph()
kg.connect()  # Neo4j 없으면 인메모리 모드

# 메타데이터에서 그래프 자동 구축
kg.build_from_metadata(metadata)

# 검색
results = kg.search_by_person("김철수")
related = kg.get_related_items(item_id)
```

### 4. storage.py (통합 저장소)

|클래스 | 기능 | 서비스 |
|---------|----------|----------|
|`PostgreSQLStorage` | 메타데이터 저장 | PostgreSQL |
|`ElasticsearchStorage` | 전문 검색 | Elasticsearch |
|`MinIOStorage` | 파일 저장 | MinIO/S3 |
|`UnifiedStorage` | 통합 관리자 | 전체 |

-----

사용 예시:
```
from src.core.metadata.storage import UnifiedStorage

storage = UnifiedStorage({
    "local_path": "./data",
    "use_local_fallback": True  # 서비스 없으면 로컬 사용
})
storage.connect_all()

# 저장
storage.save(metadata, file_path="photo.jpg")

# 검색
results = storage.search("가족 사진")
```

---

### 💡 핵심 포인트

1. **외부 서비스 없이 실행 가능**: `use_local_fallback=True` 설정 시 PostgreSQL, Elasticsearch, Neo4j, MinIO 없이도 로컬 파일로 동작

2. **전체 파이프라인 흐름**:
```
   메타데이터 생성 → 관계 추론 → 지식그래프 구축 → 저장소 저장
```

### 3.Docker로 외부 서비스 실행 (선택):
```
docker-compose up -d  # PostgreSQL, Elasticsearch, Neo4j, MinIO 실행
```
