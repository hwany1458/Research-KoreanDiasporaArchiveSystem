
### 3단계에는 다음 구성요소들이 필요합니다:

| 구성요소 | 기능 | 필요 기술 |
|---------|------|----------|
| **메타데이터 스키마** | Dublin Core 기반 + 디아스포라 확장 | Pydantic 모델 |
| **관계 추론 엔진** | 자료 간 연결 자동 생성 (얼굴, 시간, 주제, 장소) | 규칙 기반 + 임베딩 |
| **지식그래프 구축** | 노드/엣지 관계 저장 및 쿼리 | Neo4j |
| **통합 저장소** | 메타데이터, 검색, 관계, 파일 저장 | PostgreSQL + Elasticsearch + Neo4j + MinIO |

### 3단계 메타데이터 통합 계층은 현재 코드에 포함되어 있지 않습니다.
시스템 구성도를 보면 3단계에는 다음 3가지 핵심 구성요소가 있습니다:
|구성요소 | 기능 | 필요 기술 | 현재 상태 |
|--------|-----------------|-----|----------|
|메타데이터 스키마 | Dublin Core 기반 + 디아스포라 확장 필드 정의 | Pydantic | ❌ 없음 |
|관계 추론 엔진 | 인물 연결(얼굴 인식), 시간적 연결(날짜 추정), 주제적 연결(키워드), 장소적 연결 | 규칙 기반 + 임베딩 | ❌ 없음 |
|지식그래프 구축 | Neo4j 기반 노드(자료, 인물, 장소, 사건) / 엣지(depicts, relatedTo, partOf) 관계 저장 | Neo4j | ❌ 없음
|통합 데이터 저장소 | PostgreSQL(메타데이터) + Elasticsearch(검색) + Neo4j(관계) + MinIO(파일) | DB 연동 | ❌ 없음|

### 📁 구현된 클래스 및 기능

#### 1. `schema.py` - 메타데이터 스키마
| 클래스 | 기능 |
|--------|------|
| `DublinCoreMetadata` | Dublin Core 15개 핵심 요소 |
| `DiasporaExtension` | 디아스포라 확장 필드 (이주, 인물, 장소, 행사) |
| `PersonInfo` | 인물 정보 (이름, 세대, 이주 정보) |
| `LocationInfo` | 장소 정보 (좌표, 지역 분류) |
| `EventInfo` | 행사 정보 (결혼, 졸업, 명절 등) |
| `UnifiedMetadata` | 통합 메타데이터 스키마 |
| `MetadataFactory` | 자료 유형별 메타데이터 생성 팩토리 |
------

#### 2. `relation_engine.py` - 관계 추론 엔진
| 클래스 | 기능 |
|--------|------|
| `FaceRelationEngine` | 얼굴 인식 기반 인물 연결 |
| `TemporalRelationEngine` | 시간/날짜 기반 관계 추론 |
| `SpatialRelationEngine` | 장소 기반 관계 추론 |
| `TopicRelationEngine` | 키워드/의미 기반 관계 추론 |
| `EventRelationEngine` | 행사 기반 관계 추론 |
| `RelationInferenceEngine` | 통합 관계 추론 파이프라인 |
------

#### 3. `knowledge_graph.py` - 지식그래프
| 클래스 | 기능 |
|--------|------|
| `Node` | 그래프 노드 (Item, Person, Location, Event, Topic) |
| `Edge` | 그래프 엣지 (관계) |
| `KnowledgeGraph` | Neo4j 연동 지식그래프 (인메모리 폴백 지원) |
------

**주요 메서드:**
- `create_item_node()`, `create_person_node()`, `create_location_node()`
- `create_edge()`, `create_depicts_edge()`, `create_tagged_with_edge()`
- `build_from_metadata()` - 메타데이터에서 그래프 자동 구축
- `get_neighbors()`, `find_path()`, `get_related_items()`
- `search_by_person()`, `search_by_location()`, `search_by_event()`

#### 4. `storage.py` - 통합 저장소
| 클래스 | 기능 |
|--------|------|
| `PostgreSQLStorage` | 관계형 메타데이터 저장 |
| `ElasticsearchStorage` | 전문 검색 인덱싱 |
| `MinIOStorage` | 파일 저장소 (S3 호환) |
| `UnifiedStorage` | 통합 저장소 관리자 |
