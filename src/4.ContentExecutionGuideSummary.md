## 📚 4단계 실행 가이드 요약

### 1. 디지털 아카이브 API 서버 (`server.py`)

#### 서버 실행:
```
# 기본 실행
python src/core/api/server.py --host 0.0.0.0 --port 8000

# uvicorn으로 실행 (개발용)
uvicorn src.core.api.server:create_api_server --reload --port 8000
```

#### 주요 엔드포인트:

|엔드포인트 | 메서드 | 기능 |
|---------|-----------|--------|
|`/items` | GET/POST | 자료 목록/생성 |
|`/items/{id}` | GET/PUT/DELETE | 자료 CRUD |
|`/search` | POST | 전문 검색 |
|`/persons` | GET | 인물 목록 |
|`/timeline` | GET | 타임라인 |
|`/graph` | GET | 관계 그래프 | 


API 문서: http://localhost:8000/docs

### 2. 인터랙티브 스토리 생성기 (generator.py)

#### CLI 실행:

```
# 가족사 스토리
python src/core/story/generator.py --type family --output story.json

# HTML로 내보내기
python src/core/story/generator.py --type family --output story.html --format html
```

Python 사용:
```
from src.core.story.generator import StoryGenerator, StoryExporter

generator = StoryGenerator()
generator.initialize()

# 가족사 스토리 생성
story = generator.generate_family_history(items, persons, "김씨 가족")

# HTML 내보내기
html = StoryExporter.to_html(story)
```

#### 스토리 유형:

- `family_history` - 가족사
- `migration_journey` - 이주 여정
- `timeline` - 타임라인
- `interactive` - 분기형

### 3. 통합 서버 (`main.py`)
```
# API + 스토리 통합 서버
python -m src.core.api.main --port 8000
```

#### 스토리 API:

|엔드포인트 | 기능 |
|---------|-----------|
|`/stories/generate/family` | 가족사 생성|
|`/stories/generate/migration` | 이주 여정 생성|
|`/stories/generate/timeline` | 타임라인 생성|
|`/stories/generate/interactive` | 분기형 생성|

### 4. curl/Python 예제
```
# 자료 생성
curl -X POST http://localhost:8000/items \
  -H "Content-Type: application/json" \
  -d '{"title": "1975년 가족 사진", "material_type": "photograph"}'

# 검색
curl -X POST http://localhost:8000/search \
  -H "Content-Type: application/json" \
  -d '{"query": "가족 LA"}'

# 스토리 생성
curl -X POST http://localhost:8000/stories/generate/family \
  -H "Content-Type: application/json" \
  -d '{"family_name": "김씨 가족", "item_ids": ["item_001"]}'
```

### 💡 핵심 포인트

1. OpenAI API 없이도 동작: 기본 템플릿으로 스토리 생성 가능
2. Swagger UI 제공: http://localhost:8000/docs 에서 API 테스트
3. 다양한 내보내기: JSON, HTML, Markdown 지원
4. 분기형 스토리: 인터랙티브 선택지 지원
